Сервер

Можно ли использовать ноутбук как сервер?

Для разработки и тестов - да: можно запускать backend на ноутбуке и подключаться с телефона через локальную сеть или туннель. Для реального приложения лучше облако: постоянный IP/домен, SSL, резервные копии, мониторинг, хранение файлов и push-уведомления.

  • Backend: Node.js/NestJS или FastAPI, WebSocket для чатов.
  • База: PostgreSQL для аккаунтов, чатов, ролей и сообщений.
  • Файлы: S3-compatible storage с проверкой размера и типа файла.
  • Mobile: React Native/Expo или Flutter, общий API с web.
  • Push: Firebase Cloud Messaging и APNs для iOS.

Безопасные правила

Что важно заложить сразу

  • Почта не отдается в публичных API профиля.
  • Пароли хранятся только как bcrypt/argon2-хеши.
  • Приватные чаты доступны только участникам.
  • Блокировка работает в обе стороны и видна при попытке написать.
  • Флуд ограничен размером сообщения и cooldown 3-5 секунд.
  • Экспорт игры фиксирует автора-персонажа на момент сообщения.

Под вопросом

Функции для следующего слоя

Push-уведомления

Нужны для mobile и web, но требуют backend-событий, настройки FCM/APNs и пользовательских настроек тишины.

Онлайн-статус

Можно сделать опциональным: показывать всем, только игрокам/друзьям или никому.

Сноски/offtop в игре

Лучше отдельным типом сообщения или тредом, чтобы обсуждения не терялись среди игровых постов.

Три языка

Русский, украинский и английский стоит заложить через i18n-словарь с первого production-экрана.
