# Raven RP prototype

Статический web-прототип приложения для ролевых чатов под web, Android и iOS.

## Что уже показано в демо

- Авторизация по логину/паролю/почте с пояснением, что почта не публичная.
- Публичный профиль: аватар, никнейм с эмодзи, статус и описание до 500 символов.
- Три типа пространств:
  - личные заметки в формате чата;
  - игровые приватные чаты, где сообщения отправляются от лица персонажа;
  - флудовые приватные/групповые чаты, где сообщения отправляются от аккаунта.
- Визуальное разделение игровых, флудовых и заметочных чатов цветом.
- Базовое форматирование текста: жирный, курсив, подчеркивание, зачеркивание.
- Эмодзи, демо-вложения с безопасным лимитом 25 MB.
- Редактирование собственных сообщений.
- Ограничение флуда: 800 символов и cooldown 4 секунды.
- Серый стиль для сообщений вышедших персонажей/аккаунтов.
- Экспорт истории игрового чата в `.txt`.
- Демо-блокировка пользователя и поиск по логину.

## Как запустить локально

Откройте `index.html` в браузере или запустите простой локальный сервер:

```bash
python3 -m http.server 8000
```

После этого откройте `http://localhost:8000`.

## Можно ли использовать ноутбук как сервер

Для разработки - да. Backend можно поднять на ноутбуке и подключать телефоны через одну Wi-Fi сеть
или временный HTTPS-туннель. Для настоящего приложения нужен постоянный сервер или облако:

- домен и HTTPS;
- база данных с резервными копиями;
- хранение файлов;
- WebSocket/SSE для живых чатов;
- push-уведомления через FCM и APNs;
- мониторинг и ограничение нагрузки.

## Рекомендуемый production-стек

- **Web/mobile:** React + React Native/Expo или Flutter.
- **Backend:** Node.js/NestJS, FastAPI или другой framework с WebSocket.
- **База:** PostgreSQL.
- **Файлы:** S3-compatible storage.
- **Авторизация:** email verification, bcrypt/argon2 для паролей, refresh tokens.
- **Push:** Firebase Cloud Messaging для Android/web и APNs для iOS.
- **i18n:** словари для русского, украинского и английского с самого начала production-версии.
